FORMAT: 1A

# Polls

Polls is a simple API allowing consumers to view polls and vote in them.

元数据

蓝图以元数据部分开始。在这种情况下, 我们已经指定格式的值为1A。格式关键字表示 API 蓝图的版本。

api名称和介绍

蓝图中的第一个标题用作 API 的名称, 在本例中为 "民意测验"。标题以一个或多个 # 符号开头, 后跟一个标题。这里的 API 名称使用一个哈希来区分它的第一级。使用的 # 号将决定标题的级别。
下面的标题是对 API 的描述。您可以使用进一步的标题来分解描述部分。
 
资源组
 
现在是开始记录 API 资源的时候了。在标题的开头使用组关键字, 我们创建了一组相关资源。

# Group Questions

Resources related to questions in the API.

资源
 
在问题资源组中, 我们有一个名为 "问题集合" 的资源。此资源允许您查看问题列表。标题指定用于访问标题末尾的方括号内的资源的 URI。
 
## Question Collection [/questions]
 
行为
 
API 蓝图使您可以指定对资源所做的每个操作。在资源内部使用标题指定操作, 并在该操作的名称后面加上 HTTP 方法。
 
### List All Questions [GET]
 
一个操作应该至少包括一个来自服务器的响应, 其中必须包含一个状态代码, 并且可能包含一个正文。响应被定义为操作中的列表项。列表是通过前面的列表项创建的, 其中有一个 +、* 或-。

此操作将返回一个200状态代码以及一个 JSON 正文。

"接口" 资源有第二个操作, 允许您创建新问题。此操作包括一个说明, 显示要发送到服务器以执行此操作的结构。

### Create a New Question [POST]

You may create your own question using this action. It takes a JSON object
containing a question and a collection of answers in the form of choices.

+ question (string) - The question
+ choices (array[string]) - A collection of choices. 

此操作以 JSON 有效负载作为请求的一部分, 如下所示:

+ Request (application/json)

            {
                "question": "Favourite programming language?",
                "choices": [
                    "Swift",
                    "Python",
                    "Objective-C",
                    "Ruby"
                ]
            }

下一个资源是 "问题", 它代表单个问题。

## Question [/questions/{question_id}]

URI 模板

"问题" 资源的 uri 使用由 uri 模板表示的变量组件。在这种情况下, 有一个名为 question_id 的 ID 变量, 在 URI 模板中表示为 {question_id}。

URI 参数

uri 参数应使用参数列表来描述 uri。对于 "问题", 它将如下:

+ Parameters
    + question_id (number) - ID of the Question in the form of an integer

URI 模板的 question_id 变量是此资源上的每个操作的参数。它是使用任意类型号定义的, 后跟参数的描述。

Actions
